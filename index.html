The model. A connection is made once per external org — several of the same type are fine (two GitHub orgs), so a grant is two-level: which org → which resource. Each product is granted a segment (its own repo / account / space). A team sees what it was granted, requests what it's missing (org approves), and adds its own manual integrations. Governance and audit live on the org side; everyday setup happens from the product.
